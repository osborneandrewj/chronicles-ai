# v0.4.2 Patch

**Target**: 2026-05-23 (shipped on this date)
**Bar**: "`next build` succeeds in CI on Railway, and the 404/500 pages prerender without crashing."
**Budget**: ~30 minutes.

> **Backfilled doc.** Written 2026-05-25. v0.4.2 was tagged at the merge of PR #8 without a milestone doc; this file canonizes the fix in retrospect.

v0.4.2 is a deploy-only patch that unblocks Railway production builds on Next.js 15.5. Two unrelated build-time failures had to be solved before v0.4.1 could be re-shipped under the new toolchain:

1. `next build` was opening the SQLite file at module-load time and running through the auth middleware during static analysis, both of which fail in the build container.
2. Next 15.5 changed how `/404` and `/500` are prerendered, breaking the previous error-page setup.

No runtime behaviour changes for users.

## Why

1. **Build was opening the DB.** `src/lib/db.ts` was constructed at module import. During `next build`, the database file doesn't exist (different filesystem, different working dir), and the import threw, killing the build.
2. **Build was passing through middleware.** Static analysis of routes was invoking the Basic-auth gate, which rejected the build's own requests as unauthenticated.
3. **Next 15.5 prerender changed.** The previous `pages/_error.tsx` shape no longer satisfied the new prerender for `/404` and `/500`, producing a build-time crash.

## Scope

Five files. Thirty-eight lines added, one removed.

```
src/lib/db.ts            EDIT. Guard the better-sqlite3 connection behind `process.env.NEXT_PHASE !== 'phase-production-build'`.
src/middleware.ts        EDIT. Short-circuit during build phase so static analysis isn't gated by Basic-auth.
src/pages/_error.tsx     NEW. Minimal default-export _error component to satisfy Next 15.5 prerender.
src/app/not-found.tsx    NEW. Two-line not-found page so /404 prerenders cleanly under App Router.
next-env.d.ts            EDIT (auto). Next CLI added a reference for the new error types.
```

No schema, no env, no migrations.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Where to detect build phase | **`process.env.NEXT_PHASE === 'phase-production-build'`** | Next sets this during `next build`; cleanest signal that avoids leaking build awareness into runtime code paths. |
| How to disable middleware during build | **Short-circuit return at the top of the matcher function** | Modifying the `matcher` config would silently change runtime behaviour. Returning early on the phase env preserves the runtime gate exactly. |
| Whether to ship a styled error page | **No** | Minimum-viable error page that satisfies the prerender. Styling is a v0.5+ concern. |
| Whether to commit the auto-regenerated `next-env.d.ts` | **Yes** | Convention in the repo; deviating would invite noise on every `next build` locally. |

## Explicit cuts

- Styled `/404` and `/500` pages — minimum-viable is fine until the design pass.
- A general "build phase detection" helper — two call sites doesn't justify it.
- Tests for the build-phase guards — would require spawning a build process; not worth it for a defensive guard that fails loudly if removed.

## Accepted tradeoffs

- **The build-phase guards are duck-typed on an env var.** If Next renames `NEXT_PHASE` (it has changed in past major bumps), build will silently revert to the old failure mode. We will catch this on the next major Next upgrade; cheaper than building a more defensive abstraction now.
- **`src/lib/db.ts` now returns a `null`-ish handle during build.** Any code path that runs at module load and tries to query will crash. None do today; this is an enforced invariant going forward.

## Exit criteria

All confirmed at release time:

1. `npm run build` succeeds locally without `DATABASE_URL` set.
2. Railway production deploy completes through the build phase without auth or DB errors.
3. Hitting an unknown route in production returns a 404 page (any 404 page; styling not in scope).
4. `npm run lint` and `npm run type-check` pass.
5. Tag `v0.4.2` on `main`. *(Tagged at the PR #8 merge.)*
