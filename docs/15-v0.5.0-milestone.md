# v0.5.0 Milestone

**Target**: 2026-05-25 (shipped on this date)
**Bar**: "Basic-auth middleware is gone now that Cloudflare Access gates the app at the edge."
**Budget**: ~1 hour. Patch-sized.

> **Backfilled doc.** Written 2026-05-25 after the fact. v0.5.0 was tagged at `28e52c2` without a milestone doc. This file canonizes what actually shipped versus what the version number suggests.

v0.5.0 is, in practice, a small patch on top of v0.4.2: it removes the Basic-auth middleware and adds the working-autonomy notes to `CLAUDE.md`. The minor-version bump reflects the original *plan* for v0.5 — a structural shift from the JSON `turn_states` blob to typed `characters` / `places` / `scenes` tables — which was code-completed on the `chore/remove-basic-auth` branch (commit `95dbe85`) but **was not merged into `main`** at release. That work is preserved on the `save/v0.5-typed-state` branch and is queued to ship as v0.6.0.

If you came to this doc looking for the typed-state design, see [`save/v0.5-typed-state:docs/15-v0.5-milestone.md`](https://github.com/osborneandrewj/chronicles-ai/blob/save/v0.5-typed-state/docs/15-v0.5-milestone.md) on that branch. It will be ported to `docs/20-v0.6.0-milestone.md` when the work lands.

## Why

1. **Cloudflare Access is the new gate.** Once the production domain (`joinpassages.com`) was wired through Cloudflare and Access policies enforced login at the edge, the in-app Basic-auth middleware became dead weight — and a foot-gun: any future env-var drift could expose the app twice or not at all.
2. **Working autonomy needed to be written down.** Multiple sessions had been spent re-deciding "should you ask before X" for the same X. `CLAUDE.md` now states it once.

## Scope

Three files. Twenty-six lines added, sixty-six removed.

```
src/middleware.ts         DELETE the Basic-auth gate. Static export of the middleware function removed.
.claude/settings.json     NEW. Allowlist of Bash commands + reversible-action approvals.
CLAUDE.md                 EDIT. Add "Working autonomy" section.
```

No schema changes. No new env vars (though `APP_PASSWORD` becomes unused — see Loose ends).

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Where to gate auth | **Cloudflare Access at the edge** | Removes the responsibility from app code; identity is centralised; revoking access doesn't need a redeploy. |
| What to do with `src/middleware.ts` | **Delete it entirely, not stub it** | Anything left in middleware will eventually grow back. A clean removal forces a deliberate decision next time we need request-level interception. |
| What to do with the `APP_PASSWORD` env var | **Leave it set in Railway for now** | Removing prod env vars is mildly destructive and unrelated to the v0.5.0 cut. Tracked as a v0.5.2 cleanup item. |
| Whether to keep the original v0.5 typed-state plan | **Park the work, ship the auth slice only** | Two PRs were open against `chore/remove-basic-auth`; only PR #9 (auth removal) was merge-ready. Rather than block on the bigger change, we shipped the slice that was ready. |

## Explicit cuts

Everything below was scoped to v0.5 originally and **did not ship**. The work exists on `save/v0.5-typed-state` (commit `95dbe85`) and is the substance of v0.6.0:

- `characters`, `places`, `scenes` tables + v5 migration
- `src/lib/world-state.ts` (typed readers) + `src/lib/archivist.ts` (Zod patch, Haiku extractor, transactional apply)
- `/api/world-state` route + `WorldInspector.tsx` drawer
- Replay cost-accounting fix (`/api/tts/record` accepts `{ turnId, chars }`, additive `addTtsChars`)
- World-creation form fields for player character + opening place
- Vitest harness + 14 tests (migration safety, archivist patch application)
- `prompts/archivist-system.md`

## Accepted tradeoffs

- **The version-number signal is misleading.** A minor bump usually means a feature; v0.5.0 has none beyond the auth move. v0.6.0 will carry the actual feature weight. This doc exists partly to prevent future confusion about that.
- **`save/v0.5-typed-state` is parked, not maintained.** It does not get rebased onto `main` as v0.5.1 and the eventual v0.5.2 land. Whoever picks it up will resolve the accumulated drift in one pass during the v0.6.0 sprint.

## Exit criteria

All confirmed at release time:

1. `npm run lint` and `npm run type-check` pass.
2. Production deploy reachable through Cloudflare Access; un-authenticated requests are blocked at the edge.
3. No middleware in `src/middleware.ts`; build does not warn about an unused middleware export.
4. `CLAUDE.md` "Working autonomy" section present and reflects what was already understood between sessions.
5. Tag `v0.5.0` on `main` at `28e52c2`. *(Shipped — annotation is thin and should be considered superseded by this doc.)*
