# v0.6.17 — Living Momentum + a Dossier That Fills

**Status:** design approved 2026-05-30, pre-implementation
**Branch:** `v0.6.17-living-momentum`
**Author:** Andrew Osborne (with Claude)

## Problem statement

Inspection of the production world **"Joe Gallic Wars"** (world id 12, 41 turns) surfaced five issues, three of them general to the engine:

1. **Historical anachronisms** in player-visible prose: "kilometer marker" (turn 946, originated in the NPC agent's planned action and staged verbatim by the narrator), "lorica segmentata" (segmented plate is ~1st century AD; Caesar's legionaries wore mail), and a centurion ordering "two maniples" (a scale/era error). The prose quality is otherwise strong.
2. **The story dossier is completely empty** — 0 threads, 0 clues, 0 objectives, 0 timeline events — despite obvious material (an ambush threat, the birch-bark map clue, a warn-command objective). The inspector's Story tab is blank.
3. **Stories stall** unless the player actively drives them. The narrator is reactive; nothing happens *to* the protagonist unbidden.
4. **Turn numbers are global.** The single `turns` table uses a global autoincrement `id`, so this world's first turn is id 910. Those ids leak into the UI via `[t:N]` provenance tags on character facts.
5. **Stored-state cruft** specific to Joe's world: duplicate memorable_facts and the "kilometer marker" baked into a fact.

### Root cause of the empty dossier (investigation result)

The apply path (`applyArchivistPatch`) and the per-turn gate (`shouldRunArchivistLlm`) are **correct**. The Haiku archivist simply never *emits* dossier fields:

- Turn 910 (opening) ran with `isOpening: true`, which injects `OPENING_ARCHIVIST_DIRECTIVE` ("you MUST create at least one story_thread + set the place kind"). Haiku's stored patch contained **only `[characters]`** — no thread, place `kind` left null. **The bootstrap directive was ignored.**
- Every later turn emitted `[characters]` only (or was skipped on low-signal turns). Not one dossier row across 41 turns, including the combat/discovery turns.

It is a model-compliance problem: the dossier rules sit ~60 lines into an ~80-line prompt that repeatedly says "empty dossier is normal / when in doubt omit." Haiku takes the cheap path (one memorable_fact, stop). With no thread ever seeded, there is nothing to update, so it never starts.

### Two assumptions corrected during investigation

- **The narrator is Grok 4.3** (`grok-4.3`), not Sonnet. Narrator-side prompt edits target the Grok prompt; the NPC agent and archivist are Haiku.
- **Momentum and the dossier are linked.** The narrator has "the world has pulse" language and an anti-*stall* nudge, but nothing that makes events happen *to* the player, and no story clocks/threads to draw pressure from. Momentum leans on the dossier being populated.

## 1. Per-world turn numbers (display-only)

**Goal:** the first turn of a world reads as `1`, not `910`; counts shown to the user are per-world.

**Approach:** render-time mapping, no schema change.

- `buildTurnNumberMap(worldId): Map<number, number>` — uses the existing id-ordered `getTurnTimestampsForWorld(worldId)` (already `ORDER BY id ASC`); ordinal = index + 1.
- `relabelProvenance(text, map): string` — rewrites `[t:<globalId>]` → `[t:<ordinal>]`. Unknown ids (shouldn't occur) are left as-is.
- Apply at the display edge only — the `WorldInspector` surface where memorable_facts / observations render (and any visible turn counts).

**Why display-only:** storage stays append-only and global (provenance ids authored by `appendFactWithProvenance` remain stable and unambiguous); the change is fully reversible and never touches the prod `turns` table.

**Tests:** map correctness (gaps/order), relabel of single and multiple tags, no-op on text without tags.

## 2. Historical-fidelity rule (generic now, per-world profile deferred)

**Goal:** stop modern-unit / anachronistic detail in historical worlds, generically.

**Approach:** a principle-based rule added to **both** prompts:

- `prompts/narrator-system.md` (Grok narrator)
- `prompts/npc-agent-system.md` (Haiku NPC agent — the "kilometer marker" originated in its planned action)

Rule (paraphrase): *When the world's premise/region establish a historical or pre-modern setting, keep units, technology, institutions, and military organization era-appropriate. Never introduce modern measurements (e.g., kilometers), devices, or vocabulary the setting wouldn't have. When unsure of a term, choose period-neutral phrasing rather than a modern one.* One example, **no** long banned-word list (avoids over-fitting). The premise already carries the era signal into both prompts.

**Deferred:** a per-world period/tech profile (era label, measurement system, allowed/forbidden tech) seeded at world creation and injected into prompts — closer to the long-term `setting_details.existing_technology` spec. Recorded as a follow-up in the milestone doc; not built in v0.6.17.

**Tests:** prompt-snapshot assertion that the rule text is present in both prompt files.

## 3. Dossier fix (harden prompt + deterministic net)

### 3a. Prompt restructure — `prompts/archivist-system.md`

- Move the **Story Dossier** section near the top of the rules.
- Flip it from permissive to directive: *"If the latest narration introduces a danger, goal, discovery, lead, or consequence, you MUST emit the matching dossier row (thread / clue / objective / timeline). A memorable_fact is not a substitute for a thread."*
- Scope the existing "empty dossier is normal / when in doubt omit / prefer a small patch" language to **genuinely routine** turns only, so it stops licensing skips on combat/discovery turns.

### 3b. Deterministic net — `src/lib/archivist.ts` + `src/app/api/chat/route.ts`

- Split the current `OPENING_ARCHIVIST_DIRECTIVE` into:
  - a **thread-mandate** (fires whenever the world has zero active threads), and
  - a **place-kind-mandate** (true opening only).
- Add a `bootstrapDossier` path: in the chat route, compute `dossierEmpty = priorState.dossier.threads.filter(active).length === 0`. When `dossierEmpty` **and** the turn carries story signal (reuse the `richSignal` test already in `shouldRunArchivistLlm`), pass the thread-mandate directive into `extractPatch`. The bootstrap then keeps firing every meaningful turn until a thread exists — exactly the gap that left Joe's world empty (the directive previously fired only on the literal opening turn, which Haiku ignored once).
- Keep `isOpening` semantics for the true opening (place-kind mandate); introduce a distinct flag/param rather than overloading `isOpening`.
- Log the transient archivist error observed on turn 916 (one-off schema/network failure) — diagnostics only.

**Tests:** `bootstrapDossier` is true when threads empty + rich signal, false on routine turns and once a thread exists; `buildArchivistUserContent` includes the thread-mandate directive under the new flag; split directive contents.

## 4. Narrator momentum (escalating / tunable)

**Goal:** scenes don't stall waiting for the player; the world acts on its own when the player is passive — without railroading.

**Approach:** a computed momentum cue injected into `formatNarratorTurnGuidance` (same pattern as the existing anti-stall nudge), plus a narrator prompt principle.

- **Passivity detection** (`src/lib/narrator-guidance.ts`): count consecutive low-agency player moves over recent turns. "Low agency" = stance `observe` / `think`, or short continuations ("we continue", "I wait", "I look around"). Heuristics consistent with existing helpers (`isAttentionOnlyMove`, etc.).
- **Momentum ladder:**
  - **L0** — player driving → no change.
  - **L1** — 1–2 idle moves → nudge the narrator to introduce a fresh handle (existing anti-stall behavior).
  - **L2** — idle ≥ threshold, **or** an active `threat` thread carries standing pressure → **the world acts**: an NPC pursues its goal, a threat closes, time bites, a new element enters — preferring an active thread's `hidden` / stakes pressure when present. When no thread exists, the narrator may invent a proportionate intrusion consistent with the scene.
  - Thresholds are tunable constants at the top of `narrator-guidance.ts`.
- **Dossier-driven pressure:** the chat route already passes active objective/clue titles into guidance; add active **threat** threads so L2 can name the pressure. (This is why §3 ships first.)
- **Prompt addition** (`prompts/narrator-system.md`): replace the vague "The World Has Pulse" line with a Momentum subsection — *scenes must not stall; advance the world and make things happen* to *the protagonist — but create situations, not forced choices; never decide the protagonist's actions or feelings; one intrusion at a time; respect content boundaries.*

**Tests:** ladder thresholds (L0/L1/L2 selection from synthetic recent-turn sets); L2 cue references a supplied active threat thread when present; guardrail text present in prompt.

## 5. Repair "Joe Gallic Wars" (clean + backfill, backup-first)

Run against a **local copy of the prod DB first**, verify, then apply to prod (with a `.bak` snapshot on the volume before any write). All reversible; no edits to immutable turn prose.

- **Dedup** Joseph's duplicate strip-the-scout facts (`[t:934]` / `[t:936]`).
- **Scrub** "kilometer marker" from Marcus's stored memorable_fact → era-neutral phrasing. (Turn prose at 946 is append-only and stays as-is; only stored state is touched.)
- **Backfill the dossier** by re-running the hardened archivist (§3) over the transcript → seeds the ambush **threat**, the map **clue**, the warn-command **objective**, and timeline events. This validates §3 end to end.

**Done check:** the inspector Story tab for world 12 shows the seeded thread/clue/objective; facts read `[t:1…]`; no duplicate or "kilometer" fact remains.
