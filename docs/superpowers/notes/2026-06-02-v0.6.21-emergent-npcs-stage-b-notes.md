## What this is

The long-deferred **second half** of the two-part "Living NPCs" effort:
- **v0.6.18** shipped the *substrate* (rails): `npc_reveries` table, deterministic flares,
  `daily_loop`, `worldTimeBand`, routine-tick skip, plus a dormant `is_cornerstone` seam.
- **v0.6.19** was *supposed* to be the emergent layer, but only **Stage A** (3 prod bug
  fixes: anti-teleport, knowledge boundary, 4th-wall hardening) shipped under that name.
  **Stage B — the actual emergent behavior — was never built.**
- **v0.6.21 = Stage B**, the "trains on the rails." Full scope in one release (B1–B4).

Headline: NPCs **insert themselves into the player's story**, and the more the player
thwarts an NPC's authored arc, the more it stirs, acts out, and reaches in.

Predecessor spec (partly stale): `docs/superpowers/specs/2026-05-31-v0.6.19-emergent-npcs-design.md`

## Locked decisions (from 2026-06-02 brainstorming)

1. **Scope:** Full Stage B in one release — B1 dissonance→awakening engine, B2 always-on
   coloring, B3 proactive interjection (present + off-scene), B4 model escalation.
2. **Cornerstone tie-in:** Ground awakening **internally** in a cornerstone reverie (uses
   the dormant `is_cornerstone` seam) — never spoken in prose.
3. **Interjection cadence:** Conservative, matching the v0.6.20 reverie-throttle philosophy
   — per-NPC cooldown (~12 turns) + **hard cap ≤1 interjection surfaced per turn**, every
   drop logged.
4. **Awake model:** Escalate awake NPCs' ticks to **narrator-tier grok-4.3** (xai); routine
   ticks stay on Haiku.

## Cross-cutting invariant: the private-motivation rule

Dissonance, awakening, cornerstone, and interjection rationale are **internal signals
only**. They drive *behavior* and the existing private-subtext channel; the narrator must
**never name, quote, paraphrase, or explain** an NPC's motivation/awakening/memory on the
page. `npc_intents.private_rationale` stays system-only. Every STATE-block line this feature
adds is tagged `(private; render only as…)`, modeled on the current `⚡ FLARING SUBTEXT`
line (`src/lib/world-state.ts:337`). Hard rule, with its own regression test.

## Design sketch (B1–B4)

### Migration 26
- `characters.dissonance` REAL DEFAULT 0
- `characters.awakening_state` TEXT DEFAULT `'dormant'` (`dormant｜stirring｜awake`)
- `characters.last_interjection_turn_id` INTEGER (nullable)
- `npc_intents.channel` TEXT (nullable)
- Backfill existing rows: `dissonance=0`, `awakening_state='dormant'`.

### B1 — Dissonance → awakening engine (deterministic, pure, zero LLM)
- Signal from reconciler dispositions: `contradicted`/`ignored` → +dissonance;
  `staged`/`modified` → decay. Optional small loop-deviation add-on.
- Pure `nextDissonance(prev, dispositionsThisTurn, deviation) → number`, conservative
  weights + per-turn decay, clamped.
- Thresholds **with hysteresis** (separate enter/exit thresholds to prevent flapping)
  → `awakening_state`, stored on character.
- **Cornerstone grounding:** on transition *into* `awake`, if no cornerstone yet, mark the
  NPC's most-charged reverie `is_cornerstone=1`. Pure selection.
- Wire in the route right after `reconcileNpcIntentsForTurn`.

### B2 — Awakening coloring (always-on, deterministic)
- Present `stirring`/`awake` NPCs get a STATE subtext line modeled on the flare line
  (e.g. `⚡ AWAKE (private; render as visible friction/willfulness pushing slightly
  against their authored role — never name or explain it)`; `stirring` fainter).
- Cornerstone bias: awake NPC's cornerstone reverie gets raised flare priority → coloring
  rooted in that NPC's specific memory, still only via the private-subtext channel.
- Ceiling config constant the prompt enforces: color, never abandon the authored story.

### B3 — Proactive interjection (headline, agentic)
- npc-agent may emit `intent_type='interjection'` + `channel` (present: `speak`/`act`;
  off-scene: `arrive`/`phone`/`knock`/`message`).
- Plausibility: off-scene arrival rides `daily_loop` + `worldTimeBand`; an **awake** NPC may
  break routine to reach the player (the break is the drama). Narrator arbitrates — may
  `ignore`/`contradict`, raising dissonance (closes the loop).
- Knowledge: A2 boundary applies — off-scene interjector is "on comms," not present.
- Restraint: `INTERJECTION_COOLDOWN_TURNS ≈ 12` via `last_interjection_turn_id`; **hard cap
  ≤1/turn** across all NPCs; deterministic selection (awake > stirring, then higher
  dissonance); **every drop logged, never silent**.
- Surfacing: selected interjection → STATE block as a **behavioral** directive (what, not
  why); narrator writes prose; reconciler labels it next pass.

### B4 — Model escalation
- Routine ticks stay Haiku. `awakening_state === 'awake'` → that NPC's tick escalates to
  **grok-4.3** (narrator-tier).
- **New complexity vs old spec:** `runNpcAgentTick` currently batches all tickable NPCs into
  one `generateObject` call. B4 requires **partitioning** into `{awake → grok}` and
  `{rest → haiku}`, running both, merging patches (identical schema). Token usage tracked in
  `turns.metadata`.

## Cost
B1/B2 deterministic (zero tokens). B3 folds into the existing tick. Only delta is B4: awake
batch on Grok instead of Haiku, bounded by ≤1/turn cap + how many NPCs are awake. 8K narrator
budget preserved (A2 already trimmed cross-NPC belief spam).

## Testing (sketch)

  selection; soft-cap selection + drop-logging; off-scene plausibility from
  `daily_loop`+`worldTimeBand`.
- Migration 26: columns present + backfilled.
- Wiring: dispositions→dissonance on the right character; awake→Grok path; interjection→STATE
  directive.
- **Private-motivation regression:** every added STATE line carries the `(private…)` marker;
  raw `private_rationale`/`dissonance` never rendered as exposition framing.
- Browser e2e: thwart→awaken→interject (present + off-scene) within budget; inspector
  reflects state; radio-only NPC has no in-room awareness; no motivation named in prose.
