# v0.6.17 — Living momentum + dossier hardening + World 12 repair

**Target**: 2026-05-30.
**Bar**: "The world feels alive even when the player is passive — the narrator acts on its own after two idle moves, orienting turns are richly sensory, story threads actually appear, and the Gallic Wars world looks like a working chronicle instead of a broken debug artifact."
**Budget**: ~6-8 hours of focused work.

Five workstreams on top of `v0.6.16`. No new tables. Two prompt files updated, two new scripts, several data-layer and route changes. v0.6.16 is tagged but was not yet deployed to Railway; the v0.6.17 deploy will carry both milestones.

## Why

1. **The archivist wasn't seeding story threads.** Haiku was emitting only `[characters]` patches and ignoring the opening-turn directive to emit a `story_thread`. Worlds could play for many turns with an empty Story tab and no pressure on the narrator.
2. **The narrator had no momentum.** A player who typed "I look around" twice in a row got two terse reactions and nothing changed in the world. There was no mechanism to push the story forward when the player was passive.
3. **Historical-fidelity was ad-hoc.** The narrator could describe a Roman soldier checking a "kilometer marker" with no guard against anachronistic vocabulary or technology.
4. **Turn provenance was global, not per-world.** The inspector showed `turns.id` (a database-wide autoincrement), which is meaningless to a player who wants to know "how many turns into *this* world" they are.
5. **World 12 ("Joe Gallic Wars") was broken.** Duplicate facts, an anachronistic "kilometer marker" fact, and an empty Story tab left the world in a bad state that served as a misleading reference point for testing.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Idle threshold | 2 consecutive passive moves | Low enough to feel alive; high enough that the narrator doesn't wrest control on the very first quiet moment. Tunable constant. |
| Observation-move detection | Keyword heuristic on player input ("look", "observe", "survey", "glance") | No extra LLM call; false-positive cost is a slightly richer response, not a wrong one. |
| Dossier bootstrap trigger | `hasRichStorySignal(turn)` guard before forced injection | Avoids seeding a thread on purely mechanical turns (inventory checks, one-word inputs) that carry no narrative content. |
| Turn numbers display-only | Compute from `turns` table at query time; no schema column | Avoids a migration; a real `turn_number` column is tracked as a follow-up. |
| Historical-fidelity as a prompt rule | Flat prohibition in narrator + NPC prompts | Fastest guard against anachronism. A per-world period/tech profile is the robust version, deferred. |
| World 12 repaired via scripts | Standalone `scripts/` rather than a migration | One-off data repair; migration infrastructure is for schema, not content. |

## Architecture sketch

```
POST /api/chat
  │
  ├─ persist player turn
  ├─ build narrator context
  │     └─ momentum check: N consecutive passive turns?
  │           yes → inject unbidden-development directive
  │           no  → check observation-move → inject depth directive
  ├─ stream narrator response
  ├─ persist narrator turn
  ├─ run archivist
  │     └─ bootstrapDossier: no active thread + story signal?
  │           yes → force-inject story_thread patch
  └─ return stream
```

## Explicit cuts

- **Per-world period/tech profile** — deferred; the prompt rule catches the obvious cases.
- **Real `turn_number` column** — display-only for now; schema migration tracked as follow-up.
- **Story clocks as first-class persisted state** — momentum currently reads thread pressure heuristically; a `story_clocks` table is deferred.
- **Bulk world repair tooling** — scripts are world-12-specific; generalisation deferred.

## Accepted tradeoffs

- **Observation heuristic produces false positives.** A player typing "I look at the sword" gets a rich sensory beat even if they meant it as a quick glance. The cost is a slightly longer response, which errs on the side of quality.
- **Forced dossier thread may not match the narrator's intent.** The bootstrap injects a plausible thread rather than a perfect one; the archivist will refine it on subsequent turns.
- **Momentum acts at turn N+1, not mid-stream.** A passive player will get two quiet turns before the world moves. This is intentional — one passive turn should still feel like player agency.
- **World 12 scripts are irreversible without a backup.** Scripts should be run after snapshotting the DB; documented in the script header.

## Exit criteria

1. `npm run lint` and `npm run type-check` pass.
2. `npm test` passes.
3. `package.json` reads `v0.6.17` on the release branch.
4. A fresh world seeds at least one `story_thread` within ~3 story-bearing turns (visible in the Story tab of the Inspector).
5. Two consecutive passive player moves ("I wait", "I look around") produce an unbidden development on the third narrator turn — something changes in the world without the player driving it.
6. An orienting move ("I look around") returns a rich, multi-sensory establishing beat — at least a full paragraph of sensory/environmental detail, not 3 sentences.
7. Inspector provenance shows per-world `#N` (first turn = `#1`), not the raw global `turns.id`.
8. World 12 Story tab shows a populated dossier; no duplicate facts; no "kilometer" in any fact text.
9. Deploy to Railway (carries v0.6.16 + v0.6.17). Smoke test: one fresh world reaches 3 turns and shows a story thread; one orienting move returns depth; World 12 looks clean.
10. Tag `v0.6.17` on `main`.

When all hold, v0.6.17 ships. Follow-ups tracked below.

## Follow-ups (deferred)

- **Per-world period/tech profile** — seed a structured era/tech constraint at world creation time (robust version of the historical-fidelity rule).
- **Real `turn_number` column** — promote the display-only heuristic to a persisted `turns.world_turn_number` column; requires a migration and a back-fill.
- **Story clocks as first-class state** — move momentum tracking out of the heuristic and into a persisted `story_clocks` table so it survives process restarts and can be inspected.
