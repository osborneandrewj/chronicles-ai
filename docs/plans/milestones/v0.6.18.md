# v0.6.18 — Living NPCs: substrate

**Target**: 2026-05-31.
**Bar**: "NPCs stop silently forgetting who they are — their reveries are durable and flare on cue when the scene rhymes with them, and an NPC the player walks away from is still doing something sensible when they walk back."
**Budget**: ~8-10 hours of focused work.

First of a two-part effort on top of `v0.6.17`. One migration (24) adds the `npc_reveries` table and the `characters.daily_loop` column; the reverie data model moves from an overwrite-with-reinclude text column to an append-only structured table, flares are computed deterministically and enforced by a narrator rule, and NPCs gain author-once daily loops that supply off-scene continuity and let routine-only ticks skip the LLM. This is the **substrate only** — cornerstone/awakening, off-screen advancement, and proactive NPC-initiated contact are deferred to v0.6.19, which is designed to read the rails this version lays down.

## Why

1. **Reveries were silently truncated.** `characters.reveries` was a newline-text column the NPC agent overwrote each tick, told to "re-include prior reveries you want to keep" — but the agent was only shown the last 4 lines, so once an NPC had more than four, the older ones could not be re-emitted and were destroyed on the next write. The most identity-defining field was the least durable.
2. **Nobody detected the "rhyme".** A reverie is supposed to *flare* when the current scene echoes it (the Westworld mechanic the feature is named for). Detection was done by no one: the agent wrote a string, it was dumped as ambient text, and the narrator was loosely told to honor it. There was nothing measurable or reproducible.
3. **Off-scene NPCs needed an LLM tick just to stand still.** Every off-scene `nearby`/`distant` NPC required a Haiku tick purely to emit a routine `activity_append`, and re-entry could contradict what the NPC had been "doing". There was no baseline rhythm to fall back on.

## Scope

One migration, one new data module, one new time/loop helper module, NPC-agent authoring and persistence changes, STATE-block surfacing, a narrator rule, archivist character-merge changes, and the inspector. The reverie text column is left dormant (not dropped).

### Migration 24 (SQL)

```sql
CREATE TABLE npc_reveries (
  id                   INTEGER PRIMARY KEY AUTOINCREMENT,
  world_id             INTEGER NOT NULL REFERENCES worlds(id) ON DELETE CASCADE,
  character_id         INTEGER NOT NULL REFERENCES characters(id) ON DELETE CASCADE,
  text                 TEXT    NOT NULL,
  match_tags           TEXT    NOT NULL DEFAULT '',
  intensity            REAL    NOT NULL DEFAULT 0.5
                         CHECK (intensity >= 0 AND intensity <= 1),
  is_cornerstone       INTEGER NOT NULL DEFAULT 0,
  created_turn_id      INTEGER REFERENCES turns(id) ON DELETE SET NULL,
  last_flared_turn_id  INTEGER REFERENCES turns(id) ON DELETE SET NULL,
  created_at           TEXT    NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_npc_reveries_character ON npc_reveries(character_id);

ALTER TABLE characters ADD COLUMN daily_loop TEXT;
-- backfill: each characters.reveries newline → one npc_reveries row
--   (text only; match_tags ''; intensity 0.5; created_turn_id null)
```

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Reverie storage | Dedicated append-only `npc_reveries` table (not JSON, not text) | The truncation bug disappears by construction: rows are inserted and never bulk-overwritten, so the agent no longer re-emits priors at all. |
| Flare detection | Pure `computeReverieFlares` over structured scene tags | Deterministic, free, and unit-testable; flare frequency and "did prose honor it" become measurable instead of hoped-for. |
| Scene-tag sources | Structured only (place `match_tags`, dossier `relevance_tags`, occupancy hooks) — no raw player-text tokenization | Keeps the flare reproducible turn-to-turn; raw text would make it noisy and non-deterministic. |
| Flare caps | Per NPC: highest-scoring reverie with ≥1 tag overlap; global ≤2/turn, present NPCs preferred | Prevents every NPC flaring at once; keeps the signal scarce enough to stay load-bearing. |
| Daily-loop storage | `characters.daily_loop` JSON column, not a table | A loop is read as one unit per NPC; a column beats a join. |
| Loop generation | Author-once on the existing NPC-agent tick after promotion (no new call) | `local` NPCs already tick every turn; piggybacking keeps us inside the token budget and adds no hot-path latency. |
| World-time bands | New deterministic `worldTimeBand` parser, default `midday` on unparseable input | `world_time` is free-text ("Day 3, 3pm"); a defined fallback keeps loop lookup total and safe. |
| Off-scene routine | Deterministic `activityForBand` lookup; skip the Haiku tick when routine is the only update | Removes Haiku calls per turn — steady-state token cost goes *down*. |
| `is_cornerstone` column | Created now, reserved and unused | Cornerstone memory (v0.6.19) is then just a flag + a selection rule on the same table — no later migration. |
| Old `reveries` column | Left dormant, not dropped | Avoids a destructive change mid-effort; dropping it is a later cleanup. |

## Architecture sketch

```
POST /api/chat
  │
  ├─ build sceneTags  ← place.match_tags + dossier relevance_tags + occupancy hooks
  ├─ computeReverieFlares(reveries, sceneTags)
  │     └─ stamp last_flared_turn_id on chosen rows  (≤2/turn, present NPCs preferred)
  ├─ formatStateBlock
  │     ├─ flaring reverie → "⚡ REVERIE FLARING — surface as tell/hesitation/misread"
  │     └─ off-scene NPC → activityForBand(daily_loop, worldTimeBand(world_time))
  ├─ stream narrator  (honors the flaring-reverie hard rule)
  └─ NPC-agent tick
        ├─ reveries_add → INSERT npc_reveries (+ prune to ~6, evict lowest-intensity/LRU-flared)
        └─ daily_loop empty? → author one this tick (no separate call)
```

## Explicit cuts

- **Cornerstone memory + awakening/dissonance track** — `is_cornerstone` exists but unused; deferred to v0.6.19.
- **Off-screen advancement for principal NPCs** — scoped director deferred to v0.6.19.
- **Proactive NPC-initiated contact** — turn-driven intrusion deferred to v0.6.19.
- **Loop deviation/dissonance scoring** — v0.6.18 stores and *uses* loops; it does not score deviation from them (that is awakening's raw signal, v0.6.19).
- **Aligning place-population to `worldTimeBand`** — out of scope to avoid unrelated refactoring; place-population still hashes `world_time` into its PRNG seed.
- **Dropping the dormant `characters.reveries` column** — later cleanup.

## Accepted tradeoffs

- **Flares only fire on structured-tag overlap.** A reverie with no `match_tags` (e.g. every backfilled row) never flares until the agent re-tags it or adds a tagged one. This is intentional — it keeps the signal reproducible — but means existing NPCs warm up gradually rather than flaring immediately.
- **Daily loops are authored once and frozen.** Until an explicit regeneration directive, an NPC's loop does not evolve; a major life change won't reshape the routine on its own. Deviation tracking that would notice this is deferred to v0.6.19.
- **`worldTimeBand` fallback can misband odd strings.** An unparseable `world_time` defaults to `midday`, so a genuinely nocturnal scene with a weird time string may show a daytime routine line. Tolerable: the narrator can still override the band for a narrated action that turn.
- **Backfilled reveries lose nothing but gain nothing.** The migration preserves existing reverie text (intensity 0.5, no tags), so no identity is lost — but those rows are inert until re-tagged.

## Exit criteria

1. `npm run lint` and `npm run type-check` pass.
2. `npm test` passes (including the new pure-function and migration tests this version adds).
3. `package.json` reads `v0.6.18` on the release branch — bump on the release branch itself, not as a post-merge commit. The Chat header reads `pkg.version`, so any gap between merge and bump means the header lies in prod.
4. Migration 24 applied cleanly: `npc_reveries` exists and is populated by the backfill (each prior `characters.reveries` line is now a row); `characters.daily_loop` column is present.
5. In `npm run dev`, a turn is streamed in-browser where a tagged reverie **visibly flares** (⚡ in STATE) and the narrator prose honors it as subtext — a physical tell / hesitation / misread, not narrated memory-exposition.
6. An off-scene NPC's routine line appears on re-entry consistent with its `daily_loop` for the current world-time band (the narrator does not contradict where routine says the NPC is).
7. Inspector reads reveries from `npc_reveries`, showing intensity / flaring state / cornerstone fields.
8. Deploy to Railway. Smoke test: Andrew and Joe each confirm a reverie flares and honors subtext in one world, and an off-scene NPC's routine line is consistent on re-entry.
9. Tag `v0.6.18` on `main`.

When all hold, v0.6.18 ships. The emergent-behavior layer (cornerstone/awakening, off-screen advancement, proactive contact) is the next milestone, sketched in `docs/superpowers/specs/2026-05-31-v0.6.18-living-npcs-design.md` under "The v0.6.19 seam".
