# v0.2 Milestone

**Target**: 2026-06-08 (~2 weeks from 0.1.0)
**Bar**: "The narrator stays in character and on-canon through a 30-turn session, and I can see what each turn cost."
**Budget**: ~8–12 hours of focused work.

0.1.0 proved the streaming loop, authoritative state, and meta-commands. 0.2 closes the three gaps that show up immediately in real play: the narrator treats every input the same (no stance/mode), a failed stream is a dead end, and per-turn cost is captured in `turns.metadata` but invisible to the player.

Phase 1 proper (worlds, characters, scenes, archivist, name profiles) still sits behind this. 0.2 is the last quality pass on the single-world loop before we open the schema.

## Scope

Three features, all on the existing `turns` table. No new tables.

### 1. Action classifier (stance + input mode)

A Haiku-tier pre-pass that tags each player action before the narrator sees it. Output is two enums injected into the trailing user message alongside the state block.

```
stance      ∈ { do, say, think, observe, meta }
input_mode  ∈ { in-character, ooc, ambiguous }
```

The narrator prompt gets a short rubric: `do` actions narrate consequence, `say` actions render dialogue + reaction, `think` actions stay internal, `observe` actions describe without advancing time, `ooc` actions get a brief OOC reply and the scene clock does not advance.

**Why it's the biggest quality win**: "I look around" and "I attack the guard" currently get identical narrator treatment. The classifier is what stops the narrator from advancing the scene clock on observations and stops it from rendering dialogue as action.

**Files**:
- `src/lib/classifier.ts` — `classifyAction(text): Promise<{ stance, input_mode, usage }>`
- `src/lib/prompt.ts` — add stance/mode rubric to `NARRATOR_BASE`
- `src/app/api/chat/route.ts` — call classifier before `streamText`, inject into trailing user message, persist result + usage to `turns.metadata`

**Model**: `claude-haiku-4-5-20251001`. Hard 200-token output cap. Falls back to `{ stance: "do", input_mode: "in-character" }` on error — never blocks the narrator.

### 2. Retry / error UI

Right now a failed stream produces a dead UI state. 0.2 makes failures recoverable without a refresh.

- Detect stream errors in `Chat.tsx` (AI SDK `useChat` exposes `error` + `reload`).
- Show an inline error row with **Retry** and **Dismiss** buttons under the failed turn.
- On retry: re-send the same player action; the player turn is already persisted, so we re-run `streamText` against the same prefix. The narrator turn from the failed attempt is *not* persisted (already guarded by `onFinish` only running on success).
- On dismiss: hide the error, leave the player turn in place. User can type a new action.

**Files**:
- `src/components/Chat.tsx` — error state + retry/dismiss buttons
- `src/app/api/chat/route.ts` — accept a `retry: true` flag that skips re-inserting the player turn

### 3. Cost visibility

Token usage is already in `turns.metadata` for both narrator and extractor. Surface it.

- Footer strip on each assistant turn: `narrator 1,240 in / 380 out · state 410 in / 95 out · ~$0.018`.
- Top-of-page running total for the current session: `42 turns · ~$0.61`.
- Cost calc lives in `src/lib/pricing.ts` as a `const` table (Sonnet 4.6 + Haiku 4.5 rates). Re-reading the table at render time is fine; this isn't a performance path.

**Files**:
- `src/lib/pricing.ts` — rate table + `costForUsage(model, usage)` helper
- `src/components/Chat.tsx` — per-turn footer
- `src/app/page.tsx` — session total in the header

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Classifier model | **Haiku 4.5** | Two-enum classification is the canonical Haiku job. Sonnet would burn budget for no quality gain. |
| Classifier failure mode | **Fail open to `do` / `in-character`** | Never block the narrator on a classifier hiccup. Log the error in metadata. |
| Retry semantics | **Re-run against existing player turn** | Turns are append-only; we never resurrect a failed narrator turn. |
| Cost table location | **Inline `const` in `src/lib/pricing.ts`** | Pricing changes are rare and visible in git. No env var, no fetch. |
| Cost display precision | **3 sig figs, USD only** | "$0.018" reads cleanly; sub-cent precision is noise. |
| Schema changes | **None** | Classifier output and cost both ride in existing `turns.metadata`. |

## Accepted tradeoffs

- **Classifier adds latency to first token.** Haiku classification before the Sonnet stream starts adds ~300–600ms before the player sees anything. Acceptable; the narrator quality gain is worth it. Revisit if it feels sluggish.
- **Cost display is approximate.** Anthropic billing rounds differently than our `usage * rate` math. Treat the per-turn cost as a signal, not an invoice.
- **Retry doesn't recover a partial stream.** If the narrator streamed 200 tokens and then errored, those tokens are gone. Retry starts fresh.
- **Still single-world, single-session.** Everything above improves the one loop; it does not add isolation.

## Exit criteria

- Type `I look around the tavern.` → narrator describes without advancing the scene clock; metadata shows `stance: "observe"`.
- Type `/pause` → brief OOC reply, scene does not advance, no turn is persisted (meta commands bypass the classifier and DB by design).
- Kill the dev server mid-stream → UI shows a Retry button; clicking it produces a complete narrator turn against the same player input.
- 30-turn session completes with per-turn cost shown on every assistant turn and a session total in the header that matches the sum to within a cent.

When all four hold, tag `v0.2.0` and move on to Phase 1 proper per `docs/plans/roadmap.md`.
