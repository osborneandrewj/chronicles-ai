# v0.6.16 — Two-mode world creation + world archiving

**Target**: 2026-05-30.
**Bar**: "A new player can spin up a playable world in two taps — pick a genre, hit Generate — and the home screen no longer collects clutter, because finished or abandoned worlds can be archived out of sight and restored at any time."
**Budget**: ~4-6 hours of focused work.

Two independent, player-facing features shipped together on top of `v0.6.15`. No new agent, one additive migration (`worlds.archived_at`).

1. **Quick start creation mode.** `/worlds/new` gains a tab switcher. The new **Quick start** tab (default) takes only a character name (optional) and a genre picked from a 37-label grid; one Haiku call synthesizes the world (title, premise, opening location, time, character), then the existing creation tail runs unchanged (`createWorld` → `setSettingRegionForWorld` → `generateOpeningTurn` → redirect to `/play`). The **Advanced** tab is the original full form, untouched.
2. **World archiving.** A nullable `worlds.archived_at` column drives a reversible soft-hide. Active world rows gain a ⋯ menu with **Archive**; archived worlds collapse into an **"Archived (N)"** disclosure at the bottom of the home page, each restorable via **Unarchive**. No permanent delete.

## Why

1. **The Advanced form is high-friction for a first world.** Six fields including a premise paragraph is a lot to ask before someone has played once. A genre grid + name lets a player start immediately; narrative richness still arrives via the narrator's opening turn.
2. **The home list only grew.** Every experiment, test world, and abandoned run stayed visible forever with no way to tidy up. Archiving makes the home screen reflect what the player actually cares about, without destroying history.

## Scope

One additive migration. No new agent. The Quick start LLM call reuses `ANTHROPIC_API_KEY` (same provider as `region-extractor.ts`).

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Two modes, not a replacement | Quick start tab leads; Advanced preserved verbatim | Lower friction for new players without removing the control power users rely on. |
| Quick start content source | Pure LLM synthesis (Haiku, `generateObject`) | Fresh world each time; cheap and structured. Tone richness comes from the narrator's opening turn afterward. |
| Genre as a flat allowlist | `GENRES` const, server-validated via `isGenre` | Single source of truth for the picker UI and the action's input validation; rejects anything off-list. |
| Generator failure surfaces | throw → caught in action → form error | Unlike region extraction (degrades silently), a failed synthesis means nothing to create, so the player must be told. |
| Archive over delete | Nullable `archived_at` timestamp, reversible | Non-destructive; records *when* archived and orders the archived list. Matches existing `*_at` conventions. |
| Menu outside the Link | Restructured row: Link + sibling ⋯ button | A `<button>` cannot live inside an `<a>`; the menu stops propagation so it never navigates. |

## Architecture sketch

```
Quick start: name + genre
        │
        ▼
createBasicWorldAction ── isGenre(genre)? ──no──▶ form error
        │ yes
        ▼
generateWorldFromGenre(genre, name)  (Haiku, GeneratedWorldSchema)
        │  (throws → caught → "Couldn't generate a world")
        ▼
createAndOpenWorld(input)  ── shared tail ──▶ createWorld → setSettingRegion → openingTurn → redirect /play
        ▲
Advanced: full form ──▶ createWorldAction ──┘

Archiving:
  row ⋯ → archiveWorldAction(id) → archived_at = now → revalidatePath('/')
  listWorlds (archived_at IS NULL)        → active list
  listArchivedWorlds (archived_at NOT NULL) → "Archived (N)" disclosure
  archived row ⋯ → unarchiveWorldAction(id) → archived_at = NULL → back to active
```

## Explicit cuts

- **Permanent deletion** — deferred; archiving is non-destructive only for now.
- **Per-genre hand-authored templates** — Quick start is pure LLM synthesis.
- **Editing an existing world's premise/fields** — out of scope.
- **Bulk archive/unarchive** and **archiving from the play page** — home-page, one-at-a-time only this version.

## Accepted tradeoffs

- **Quick start adds one Haiku call before the world exists.** Cheap, but it is a synchronous step on the create path; a flaky call surfaces as a form error rather than a half-made world.
- **Generated premises vary in quality.** Haiku occasionally produces a thin premise; the narrator's opening turn compensates, and the player can always fall back to Advanced.
- **Soft-hide keeps all rows.** Archived worlds still consume DB space — acceptable, since restore is the whole point and there is no delete yet.
