# v0.6.5 Milestone

**Target**: TBD (after v0.6.4 ships)
**Bar**: "The play view is comfortable on a phone — composer, inspector drawer, and per-turn footer all work one-thumbed. Replaying an old narrator turn updates that turn's cost footer, not the latest one. Narrator turns vary in length with the weight of the move — small beats stay terse, entrances and consequences earn paragraphs. The narrator stays inside the protagonist's perceptual frame — no cuts to off-scene action."
**Budget**: TBD — size after the mobile design call lands.

v0.6.3 ([`docs/plans/milestones/v0.6.3.md`](v0.6.3.md)) shipped the Safari/WebKit audio fix as a focused patch, and v0.6.4 ([`docs/plans/milestones/v0.6.4.md`](v0.6.4.md)) took the story-dossier memory work. v0.6.5 picks up the two UI workstreams that were bundled with the audio fix in the original v0.6.3 scope: the mobile + desktop UI pass and the per-turn cost-footer bug. No new tables, no new agents, no new env vars.

(Character canonicalization moved to v0.6.6 — see [`docs/plans/milestones/v0.6.6.md`](v0.6.6.md).)

## Why

Three motivating problems:

1. **The play surface assumes a desktop browser.** Andrew + Joe play on phones; the inspector drawer, chat composer, and per-turn footer all need a mobile pass. Desktop also has small comfort wins worth bundling.
2. **Replaying an old narrator turn doesn't update that turn's cost footer.** v0.6.0 wired `addTtsChars(turnId, chars)` to make replay-of-old-turn additive on the right row, but the smoke test showed the per-turn footer next to the replayed turn does not move. The audio plays (once v0.6.3 lands), the cost stays wrong. Low priority; visible to anyone using the replay control. See [[v0_6_0_replay_cost_bug]].
3. **Narrator turns don't vary in length with fictional weight.** Observed pattern in current play: small thinks, location entries, and routine actions all collapse to a single 2–3 sentence paragraph. The prompt already says entrances/major transitions may expand to 2–4 paragraphs (`prompts/narrator-system.md:34`), but the model is settling into a uniform mid-length. Visible in a four-turn sequence where "I get in my car and drive to a club" and "I go into the club" — both new-location moves — both render at the same length as "I think about different places I can go".

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Bundle UI + replay-cost-footer + narrator-length in one tag | **One tag (v0.6.5)** | A and B both touch `Chat.tsx`. C is prompt-only, isolated, and low-risk — bundling avoids a separate tag for a one-file prompt edit. If the mobile design call drags, B and C can ship as a v0.6.5-patch without A. |
| Narrator-length fix shape | **Tighten triggers + add contrastive examples** | Existing rules are correct but descriptive; making them prescriptive with explicit triggers and short-correct vs short-collapsed pairs is the lowest-risk lever. Rejected: a hard "diet check / pattern-break" rule (risks forced variety) and a full restructure of the length section (risks disturbing behaviors that currently work). |
| Mobile inspector pattern | **TBD — default lean: full-screen sheet under 640px, slide-over above** | Chat is the primary surface on mobile; inspector is occasional. A side drawer on a 375px viewport leaves <200px of chat, which is unusable. Slide-up sheet keeps the chat present underneath. Pending design call. |
| Target devices | **TBD — proposed: iPhone SE (375px) min, latest iPhone Pro Max as upper, landscape supported but not optimised** | Andrew + Joe both iPhone. Android testing is out of scope until there's an Android user. |
| Replay-cost diagnostic path | **Start at `useNarratorAudio.ts` `replay()`, then `Chat.tsx` `reportTtsChars`, then DB row** | Order matches [[v0_6_0_replay_cost_bug]] hypothesis ranking. Verifying DB row updates separates client-state bugs from callback-wiring bugs. |
| Client-side cost state shape | **TBD pending diagnosis** | If the DB updates correctly and only the UI lags, change the cost-state structure to be keyed by turn id rather than "latest assistant turn". Defer the call until we know which hypothesis is right. |
| Mobile design dependencies | **No new UI library** | shadcn/ui patterns continue; sheet/drawer hand-rolled. Cost: more code. Benefit: no dependency creep for one component. Revisit if a second drawer surface appears. |

## Open questions

- **Mobile target devices:** confirm iPhone SE as the floor. Landscape — must-work or nice-to-have?
- **Inspector at narrow widths:** full-screen sheet, bottom drawer, or hidden behind a route push? (Leaning sheet.)
- **Composer behaviour with software keyboard:** does the inspector close when the keyboard opens, or stay mounted underneath?
- **Per-turn footer at <360px:** wrap to two lines, hide the per-turn cost behind a tap, or shrink the text? Each has a cost.
- **Replay-cost root cause:** which of the three hypotheses in [[v0_6_0_replay_cost_bug]] is it? Diagnose before committing to a fix shape.
- **Narrator length — does prompt tightening hold?** Prompt-only fixes for length behavior can decay as the model regresses to the mean. If a tightened prompt doesn't stick after two play sessions, escalate to a structural lever (per-stance length budget table, or post-stream length check) in a follow-up.

## Explicit cuts

- **No new agents, no new tables, no new env vars.** v0.7 territory.
- **No Android-specific work.** Until there's an Android user.
- **No PWA install / offline.** Out of scope; mobile-web is fine for v0.6.5.
- **No keyboard shortcuts pass for desktop.** Separate concern; revisit when a power-user workflow demands it.
- **No virtualised chat history.** Pagination cutoff stays; virtualisation parked until the 60-turn initial load feels slow.

## Accepted tradeoffs

- **Hand-rolled mobile sheet means more JSX in `WorldInspector.tsx`.** Tolerable for one surface; would push us to a library if a second arrives.
- **No automated viewport tests.** Mobile QA is manual on real devices. CI snapshot tests on a mobile viewport were considered and rejected — they catch layout regressions but not touch ergonomics, and the maintenance cost is high for a two-person team.
- **Landscape iPhone is "supported but not optimised".** A landscape user on a Pro Max will see desktop-shaped chrome on a short viewport. Accepted because the primary use case is portrait.
- **Replay-cost fix only addresses the per-turn footer.** The session total in the header is already correct via `addTtsChars`; if it isn't, that's a separate bug to chase.
- **Narrator length is evaluated by manual play, not automated metric.** No turn-length distribution dashboard, no length-vs-stance correlation check. The exit criterion is a played sequence Andrew can eyeball — same QA shape as every other narrator prompt change.

## Exit criteria

*(Mobile criteria sharpen after the design call.)*

1. `npm run lint` and `npm run type-check` pass. ✅
2. `npm test` passes (95/95 — no new tests added; see "Tests added" note above). ✅
3. `package.json` reads `0.6.5` on the release branch (bumped on the branch itself per [[feedback_release_version_bump]]). ✅
4. Manual (replay-cost): replay an old narrator turn (not the latest). Per-turn footer next to that turn grows by the replay's char count; session total in the header grows by the matching cost. Replay the latest assistant turn — its footer also grows, no cross-turn leakage.
5. Manual (mobile chat): on iPhone (375px) → open a world → compose and send a turn → narrator response renders without horizontal scroll, per-turn footer is legible, composer remains reachable above the software keyboard.
6. Manual (mobile inspector): on iPhone (375px) → toggle inspector → bottom sheet rises over the chat with a drag-handle + scrim → close affordance is thumb-reachable → switching Now/Story/Wiki tabs swaps the body cleanly → Wiki sub-tabs (Characters/Places/Scenes) work → reopen shows current state.
7. Manual (desktop regression): on a 1440px viewport, all v0.6.3 behaviour still works — inspector drawer, audio toggle, slash command menu, per-turn footer, audio playback.
8. Manual (narrator length): play a sequence including (a) a small think, (b) a small say, (c) an entry into a new location, (d) a routine continuation. The think and routine continuation come back in 1–3 sentences; the new-location entry expands to 2–4 paragraphs with sensory texture for the new place. Adjacent turns do not all share the same paragraph count.
9. Manual (narrator POV): with at least one off-scene agent NPC carrying recent `activity` (e.g. an NPC heading somewhere), play 3–5 turns and verify no narrator response contains a paragraph rendering off-scene action live. Off-scene NPCs surface only through in-frame means (phone, arrival, message, knock).
10. Deploy to Railway → Andrew + Joe each play one turn on phone and one on desktop. No 5xx. Inspector usable on both.
11. Tag `v0.6.5` on `main`.
